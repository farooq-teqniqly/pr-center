PR-Center · UX mockups · v1

The review inbox, four screens

One question drives every pixel: which of my open reviews changed since I last looked? Amber is spent on exactly that signal and nothing else. Reviewer states get semantic colors; everything the app merely knows (timestamps, numbers, tokens) is set in mono so data reads as data.

01

Unlock

First screen after container start. The app password derives the token-decryption key, so no polling happens until this succeeds — the screen says so instead of pretending data is fresh. No recovery path, by design; the reset wipes tokens.

localhost:8080/unlock

PR-Center is locked

Enter your app password to decrypt the stored GitHub tokens and start polling. Polling has been paused since the container started.

Forgot it? There is no recovery — and re-enter your PATs.
02

Review inbox — the main screen

Grouped by org, then repo. Unseen-first inside each group. An amber stripe + badge means someone else committed, commented, or reviewed since I last looked — my own activity never lights it. The right column answers the three time questions per PR. Clicking a title opens GitHub and marks the PR seen via a fresh live fetch. The ps-unite chip and banner show the per-owner error model: stale data stays visible, the failure is labeled.

04

Settings — tokens and polling

One fine-grained PAT per owner, pasted in-app (never env vars or compose files), shown only as a masked fingerprint after save. Status column repeats the per-owner fetch state so a bad token is diagnosed where it is fixed. The reset action is the "forgot password" path.

localhost:8080/settings
PR»Center

GitHub tokens

One fine-grained PAT per owner, with read access to pull requests on that owner's repos. Tokens are encrypted at rest with your app password. For SSO orgs, the token must be authorized for the org or its queries silently return nothing.

Polling

Poll GitHub every minutes. Each poll costs one search per owner plus per-PR detail calls.

Reset stored tokens

Deletes all encrypted tokens and the app password. Use this if you forgot the password — tokens are unrecoverable without it. You will re-enter all PATs.

Interaction notes: rows are not expandable in v1 — everything needed for the skip/review decision is on the row. No approve/comment actions exist anywhere; the only mutation the app ever performs is recording "I looked." Covered PRs are flagged, never hidden or moved. Approved PRs silently leave the list on the next poll.
